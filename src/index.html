Easy to learn

Open in DartPad {% prettify dart %} import 'dart:html'; main() async { var countdown = querySelector("#countdown"); for (int i = 100; i >= 0; i--) { countdown.text = "Time: $i"; await window.animationFrame; } } {% endprettify %}

With a consistent language, well-crafted standard libraries, and cleaner DOM, Dart is a good choice even for programmers with limited or no JavaScript experience. Things work the way C, C#, ActionScript, and Java developers have come to expect.

Dart has one way to manage dependencies, and one way to import them. Its standard libraries have many classes that would otherwise had to be provided by micro-libraries or built from scratch. This makes codebases consistent, with one preferred way to do most things.

Optimizing compilation, tree shaking, minification, and asset transformation are all baked into one build step, configured by one file.

Made for application programming

{% prettify dart %} void highlight(ParagraphElement focus, Element container) { var all = container.querySelectorAll("p"); all.forEach( (paragraph) => paragraph.classes.remove("highlight")); focus.classes.add("highlight"); } {% endprettify %}

Client-side development has challenges that are different from, say, backend development. Dart recognizes that. Client apps have a lot of asynchronicity, so Dart has async/await. Clients have many streams (of events), so Dart has a well crafted Stream class and generators. Object-oriented programming is a great fit for UI development, and Dart is purely object-oriented.

Comes with batteries included

Code completion, refactoring, step-by-step debugging, powerful static analysis, profiling, code coverage, a standard unit test library—all these are supported by the preferred IDE, IntelliJ WebStorm. Another IDE with great Dart support is the open-source Atom editor. Plugins also exist for Sublime Text 3, Emacs, and Vim.

The Dart language and tools help you find bugs early on, before they can become big problems.

And then there are the standard libraries. For example, dart:html is a sane, Dartisan wrapper around the DOM and window APIs, so you don't need to worry about browser support.
